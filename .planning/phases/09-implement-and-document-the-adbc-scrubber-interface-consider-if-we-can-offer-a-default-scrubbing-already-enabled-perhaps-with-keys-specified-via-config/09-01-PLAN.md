Add after existing ini keys:
```python
parser.addini(
    "adbc_scrub_keys",
    help="Space-separated param key names to auto-redact from recorded cassettes. Example: token password api_key",
    type="string",
    default="",
)
```
For per-driver form, pytest does NOT support dotted ini keys natively — use a naming convention:
`adbc_scrub_keys` = global keys (space-separated)
Per-driver: read via `config.getini(f"adbc_scrub_keys_{driver_name.replace('.', '_')}") `
BUT: pytest addini requires pre-registration of each key name, and driver names are dynamic.
ALTERNATIVE: Use a single `adbc_scrub_keys` key that accepts BOTH global and per-driver lines in linelist form:
```
adbc_scrub_keys =
    token password
    adbc_driver_snowflake: account_id warehouse
```
DECISION FROM CONTEXT.md: "Global form: adbc_scrub_keys = token password api_key" and "Per-driver form: adbc_scrub_keys.adbc_driver_snowflake = account_id warehouse"
Since pytest addini cannot register dynamic per-driver keys, use a DIFFERENT approach:
Register `adbc_scrub_keys` as type="linelist" to handle multiline; parse lines for `driver:` prefix.
OR: Register only `adbc_scrub_keys` as type="string" (space-separated global keys) and handle per-driver by parsing CONTEXT-specified dotted key approach differently.

IMPLEMENTATION DECISION: Register `adbc_scrub_keys` as type="linelist" where each line is either:
- `key1 key2 key3` (global keys — space-separated on one line)
- `driver_name: key1 key2` (per-driver keys — colon-separated)

This is the most pytest-idiomatic approach for dynamic per-driver config. Example pyproject.toml:
```toml
[tool.pytest.ini_options]
adbc_scrub_keys = ["token password", "adbc_driver_snowflake: account_id"]
```
Alternatively, per CONTEXT.md dotted syntax `adbc_scrub_keys.adbc_driver_snowflake`, register each known driver as a separate ini key on demand using `addini` per driver — but that requires knowing drivers upfront, which we don't.

FINAL DECISION: Use type="linelist". Parse each line: if it contains ":" → per-driver; else → global.
Build `scrub_keys_global: list[str]` and `scrub_keys_per_driver: dict[str, list[str]]`.
Pass `scrub_keys_global` and `scrub_keys_per_driver` to ReplaySession.
