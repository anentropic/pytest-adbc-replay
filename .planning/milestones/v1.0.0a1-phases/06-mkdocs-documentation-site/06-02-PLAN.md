    **Step 1 — Install the plugin and driver**
    ```bash
    pip install pytest-adbc-replay adbc-driver-duckdb
    ```
    Brief explanation: the plugin integrates with pytest; the DuckDB driver is a local in-process database requiring no credentials.

    **Step 2 — Write conftest.py**
    Show the minimal conftest that wraps a DuckDB connection with `adbc_replay.wrap()`:
    ```python
    import adbc_driver_duckdb.dbapi as duckdb
    import pytest

    @pytest.fixture(scope="session")
    def db_conn(adbc_replay):
        with duckdb.connect() as conn:
            yield adbc_replay.wrap(conn)
    ```
    Explain what each part does: `adbc_replay` is the plugin fixture (session-scoped), `.wrap()` intercepts cursor calls.

    **Step 3 — Write a test**
    ```python
    import pytest

    @pytest.mark.adbc_cassette("first_query")
    def test_first_query(db_conn):
        with db_conn.cursor() as cur:
            cur.execute("SELECT 42 AS answer")
            row = cur.fetchone()
            assert row == (42,)
    ```
    Explain: the marker sets the cassette name; without it the name is derived from the test node ID.

    **Step 4 — Record the cassette**
    ```bash
    pytest --adbc-record=once
    ```
    Explain what `once` does: records if the cassette directory doesn't exist yet. Show expected output (test passes, cassette written).

    **Step 5 — Inspect the cassette files**
    Show the resulting directory structure:
    ```
    tests/cassettes/
    └── first_query/
        ├── 000.sql      # normalised SQL — human-readable
        ├── 000.arrow    # query result as Arrow IPC
        └── 000.json     # parameters (null here — no parameters used)
    ```
    Show what `000.sql` contains: `SELECT\n  42 AS answer`. Explain the git diff angle: because `.sql` files are plain text, query changes show up as readable diffs in pull requests.

    Explain that the reader should commit `tests/cassettes/` to version control.

    **Step 6 — Replay without the database**
    ```bash
    pytest
    ```
    Explain: default mode is `none` (replay only). No driver call is made. The test passes from the cassette alone.
