    **`docs/src/reference/fixtures.md` — Fixtures**
    Document each fixture as a section:

    *`adbc_replay`*
    - Scope: session
    - Type: `ReplaySession`
    - Purpose: wraps ADBC connections to intercept cursor calls for recording or replay
    - Interface: `adbc_replay.wrap(conn)` — accepts an ADBC connection, returns a wrapped connection
    - Override: override this fixture in `conftest.py` only if you need to customize `ReplaySession` initialization (rare)
    - Example: minimal conftest pattern (same as Tutorial)

    *`adbc_scrubber`*
    - Scope: session
    - Type: `Callable[[dict | None], dict | None] | None`
    - Default: `None` (no scrubbing)
    - Purpose: called before parameter values are written to the `.json` cassette file
    - Override: return a callable from your conftest fixture override
    - Signature: `scrub(params: dict | None) -> dict | None`
    - Note: applies to parameters only, not to Arrow result data

    *`adbc_param_serialisers`*
    - Scope: session
    - Type: `dict[type, Callable] | NO_DEFAULT_SERIALISERS`
    - Default: built-in serialisers for standard Python types
    - Purpose: extend or replace JSON serialisation for parameter types not handled by default
    - `NO_DEFAULT_SERIALISERS` sentinel: when returned, disables all built-in serialisers
    - Override: return a dict mapping `type -> serialiser callable` from your conftest fixture

    **`docs/src/reference/markers.md` — Markers**

    *`@pytest.mark.adbc_cassette`*
    - Arguments: `name: str`, `dialect: str | None = None`
    - Effect: sets the cassette directory name for the decorated test to `name` (instead of the auto-derived node ID)
    - `dialect`: overrides the global `adbc_dialect` ini setting for SQL normalisation in this test
    - Cassette path: `{adbc_cassette_dir}/{name}/`
    - Note: if `name` is not provided, cassette name is derived from the test node ID (module path + test function name)
    - Auto-derived naming example: `tests/cassettes/tests__test_example__test_my_query/`

    **`docs/src/reference/exceptions.md` — Exceptions and Warnings**

    *`CassetteMissError`*
    - Module: `pytest_adbc_replay`
    - Raised when: a query is executed in `none` mode and no matching interaction is found in the cassette
    - Message includes: the normalised SQL, available interactions in the cassette
    - Importable via: `from pytest_adbc_replay import CassetteMissError`

    *`NormalisationWarning`*
    - Module: `pytest_adbc_replay`
    - Raised when: sqlglot cannot parse the SQL and the plugin falls back to whitespace-only normalisation
    - When you see this: check that the SQL dialect setting matches your database's SQL dialect; the query may still replay correctly if whitespace normalisation is sufficient
    - Importable via: `from pytest_adbc_replay import NormalisationWarning`

    **`docs/src/reference/cassette-format.md` — Cassette File Format**
    Structure section: show the directory layout with real file names:
    ```
    tests/cassettes/
    └── {cassette_name}/
        ├── 000.sql      # Interaction 0: normalised SQL
        ├── 000.arrow    # Interaction 0: query result as Arrow IPC
        ├── 000.json     # Interaction 0: parameters and driver options
        ├── 001.sql      # Interaction 1 (if test executed two queries)
        ├── 001.arrow
        └── 001.json
    ```

    File descriptions:
    - `.sql` — canonicalised, pretty-printed SQL (using sqlglot). Human-readable. Committed to version control so query changes appear as diffs in pull requests.
    - `.arrow` — query result serialised as Arrow IPC using `RecordBatchFileWriter`. Schema metadata is preserved. Language-agnostic (any Arrow library can read it).
    - `.json` — parameters passed to `execute()` and driver options. `null` when no parameters were used. Stored after scrubbing (if a scrubber is registered).

    Versioning note: cassette format is not versioned in v1.0.0. Do not modify cassette files by hand.
