Update `.github/workflows/pr.yml` to add a `test:` job that calls the reusable workflow, alongside the existing `coverage:` job. The coverage job runs on a single Python version — keep it as-is.

Replace the entire file with:

```yaml
name: PR

on:
  pull_request:

concurrency:
  group: pr-${{ github.ref }}
  cancel-in-progress: true

permissions:
  pull-requests: write
  contents: read

jobs:
  test:
    name: Test matrix
    uses: ./.github/workflows/_test.yml

  coverage:
    name: Report test coverage
    runs-on: ubuntu-latest
    timeout-minutes: 10
    steps:
      - name: Checkout code
        uses: actions/checkout@v6

      - name: Set up uv
        uses: astral-sh/setup-uv@v7
        with:
          enable-cache: true
          cache-dependency-glob: "uv.lock"

      - name: Sync dependencies
        run: uv sync --locked --dev

      - name: Run tests with coverage
        run: uv run pytest --cov=src/pytest_adbc_replay --cov-report=xml --cov-report=term

      - name: Post coverage comment
        uses: MishaKav/pytest-coverage-comment@v1.1.51
        with:
          pytest-coverage-path: coverage.xml
          github-token: ${{ secrets.GITHUB_TOKEN }}

      - name: Cache pruning
        if: always()
        run: uv cache prune --ci
```

Key points:
- Trigger is `pull_request` (unchanged)
- Add `test:` job calling `./.github/workflows/_test.yml` (no `with:` needed)
- Preserve `coverage:` job exactly as before (single Python version for coverage reporting is fine)
- The two jobs run in parallel — no `needs:` dependency between them
